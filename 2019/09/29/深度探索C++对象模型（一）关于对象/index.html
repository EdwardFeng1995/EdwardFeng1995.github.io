1.1 C++对象模式

C++对象模型

在C++中，有两种class data member：static和nonstatic，以及三种class member functions：static、nonstatic和virtual。

对于下面这个class Point声明：

Nonstatic data members 被配置于每一个class object之内，static data members 则被存放在个别的class object之外。Static 和 nonstaic function members 也被放在个别的class object 之外。Virtual functions 则以两个步骤支持：

  • 每一个class产生一堆指向virtual functions 的指针，放在虚函数表（virtual table，vtbl）。
  • 每一个class object被安插一个指针，指向相关的virtual table。通常这个指针被称为vptr

这个模型的主要优点在于它的空间和存取时间的效率；主要缺点则是，如果应用程序代码本身未曾改变，但所用到的class objects的 nonstatic data members有所修改，那么哪些应用程序同样得重新编译。

加上继承

C++支持单一继承，也支持多重继承，继承关系也可以指定为虚拟。

在虚拟继承的情况下，base class不管在继承串链中被派生多少次，永远只会存在一个实例。

base class table模型：

base class table表中的每一个slot内含一个相关的base class地址。每一个class object内含一个bptr，它会被初始化，指向其base class table。

  • 缺点：由于间接性而导致的空间和存取时间上的额外负担。

  • 优点：

    • 在每一个class object中对于继承都有一致的表现方式：每一个class object中对于继承都有一致的表现方式：每一个class object都应该在某个固定位置上安放一个base table指针，与 base classes的大小或个数无关。
    • 无须改变class objects本身，就可以放大、缩小，或更改base class table。

C++程序设计范式有三种：程序模型、抽象数据模型（ADT）、面向对象模型（OO）。

在OO范式中，原则上，被指定的object的真是类型在每一个特定执行点之前，是无法解析的。在C++中，只有通过指针和引用的操作才能完成。相反地，在ADT范式中，程序员处理的是一个拥有固定而单一类型的实例，它在编译时期就已经完全定义好了。

C++以下列方法支持多态：

  • 经由一组隐式的转化操作。例如把一个派生类指针转化为一个指向其public base type的指针。
  • 经由虚函数机制
  • 经由dynamic_cast和typeid运算符。

多态利用虚函数机制，在执行期根据object的真正类型解析出到底是哪一个函数实例被调用。这不只使得“当类型有所增加、修改或删减时，我们的程序代码无须改变”，而且也使一个心得Library_materials subtype的供应者不需要重新写出“对继承体系中的所有类型共通”的行为和操作。

一个class object需要多少内存，一般而言：

  • 其nonstatic data members的总和大小。
  • 加上任何由于alignment（将数值调整到某数的倍数）的需求而填补上去的空间。
  • 加上为了支持virtual而由内部产生的任何额外负担。

一个指针，不管它指向哪一种数据类型，指针本身所需的内存大小是固定的。

指针的类型

指向不同类型的指针间的差异：

既不在其指针表示不同，也不在其内容（代表一个地址）不同，而是在其所寻址出来的object类型不同。也就是说指针类型会教导编译器如何解释某个特定地址中的内存内容及其大小。

一个类型为void*的指针只能够持有一个地址，而不能够通过它操作所指的object。

转换（cast）其实是一种编译器指令。大部分情况下它并不改变一个指针所含的真正地址，它只影响被指出的内存的大小和其内容的解释方式。
